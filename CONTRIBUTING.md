# Contributing to EBAN-STACK

Thank you for your interest in contributing to EBAN-STACK. Please follow these guidelines.

Coding standards
- Use clear, small commits with descriptive messages.
- Follow PEP8 style. Running `black` is recommended but optional.
- Add type hints and docstrings for public functions.

Testing
- Add unit tests to `tests/` for new functionality.
- Run `pytest -q` before submitting a PR.

Pull request checklist
- [ ] Is the code properly formatted and linted?
- [ ] Are tests added/updated and passing?
- [ ] Is documentation updated (README/CHANGELOG)?
- [ ] Does the PR include a short description of the change and the reason?
